UNRAID

My homemade NAS with parity, docker, VMs and gameservers

The hardware

Currently the hardware used is all old hardware used for old computer builds. I am planning on upgrading it to more efficient and faster components.
  • Motherboard: MSI B760M Mortar
  • Processor: Intel core I5-13500
  • RAM: 32B DDR5
  • Network interface: Onboard 1Gbps ethernet port.
  • PSU: 650W
  • UPS: 1200W
  • Storage:
  • LTS: 3x 4TB NAS HDD (1 parity & 2 storage)
  • Cache: 1x 1TB SATA SSD for caching.

The software

UNRAID is an operating system that is designed to run on bare hardware, this is how I run it. It is best to not run this in a VM but it is possible. UNRAID boots of a USB drive but once loaded runs off of the RAM.

What I use it for:

Network Attached Storage

The primary use of my UNRAID machine is being a NAS. Since picture and video quality is getting better every year the file sizes have been growing just as fast. Increased file sizes, unloading the storage from my main systems and a fear of losing files when a drive fails made me build the NAS. This allows me to still move files to it from my main system just like you would on any normal folder. It also allows any other system on the local network once logged in to acces files.

Docker

I run a lot of docker containers for ease of use. A couple examples and their use cases:
- NGINX and nginx proxy manager (NPM)
To host this and some other websites I use nginx. To send visitors to the correct website and docker container I use NPM to route the incomming traffic. Inside NPM you can add domains and the location they need to point to, like a local IP and port.
- Cloudflare tunnel
To make sure there is a secure connectiong without portforwarding I use a Cloudflare tunnel. This will send users that navigate to this website to my UNRAID box where NPM will send it to the correct container.
- Nextcloud
My Nextcloud also runs in docker. You can read more about this on the dedicated page for Nextcloud.
-Game servers
For the occasional time my friends would like to play a multiplayer game in a private enviroment I host them and many of them through Docker. To allow people from anywhere to access my servers I route it via cloudflare DNS pointing it towards a DDNS that points to my home IP adress. The SRV record including the appropriate port points towards the CNAME record which points towards the DDNS.

Virtual Machines

I used to have a couple Linux VMs with Ubuntu and other Distros setup. Since this I moved those to dedicated machines to give all machines more performance. Currently the only VM is for HomeAssitant OS. This is because this will allow for all the HomeAssistant features in one central place without additional docker containers.

If you want to read more about the benefits of HA OS instead of a docker container I recommend reading this: HA installations.
